# 技法蒸馏方法

状态：活动  
目的：把公开材料转为可执行、可验证、可组合的叙事 skill

## 标准字段

每条技法蒸馏记录必须包含：

- `source_summary`：来源的短摘要，不保存原文。
- `observed_pattern`：观察到的叙事现象。
- `abstract_principle`：去作者化后的高层原则。
- `usable_skill`：可执行的技法动作。
- `forbidden_copy_boundary`：不可复制边界。
- `story_graph_fields_affected`：影响的 `story_graph` 字段。
- `validation_questions`：验收问题。
- `repair_actions`：失败修复动作。

## 蒸馏流程

1. 读取合法公开来源，只记录标题、URL、短摘要。
2. 从来源中抽象结构、节奏、钩子、递进、反转、视角、情绪或世界观压缩。
3. 删除作者姓名、作品可识别细节、原文句式和画面风格。
4. 转写为 `story_graph` 字段。
5. 写入验收问题和硬失败。
6. 通过版权边界巡检后，才允许进入技法模块。

## 抽象质量标准

- 可组合：能和其他技法共同作用于一页或一组边。
- 可验证：能用问题、分数或硬失败判断是否成立。
- 可修复：失败后有明确补救动作。
- 去作者化：最终使用者不需要知道来源作者也能执行。

## 示例

| 字段 | 示例 |
| --- | --- |
| `source_summary` | 公开访谈提到恐怖可来自日常观察。 |
| `observed_pattern` | 普通物件先保持正常，再出现一次轻微违和。 |
| `abstract_principle` | 先建立熟悉，再污染熟悉。 |
| `usable_skill` | 为每页选择一个日常物件，写入异常化阶段。 |
| `forbidden_copy_boundary` | 不复刻具体作品中的怪物、场景、画风或句子。 |
| `story_graph_fields_affected` | `suspense_type`, `rule_pressure`, `visible_evidence` |
| `validation_questions` | 读者能否说出哪里不对劲？ |
| `repair_actions` | 增加物件正常状态、第一次异常证据和回收点。 |
