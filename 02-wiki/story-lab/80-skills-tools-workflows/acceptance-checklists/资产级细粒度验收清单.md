# 资产级细粒度验收清单 / Asset-Level Fine-Grained Acceptance Checklist

## Purpose / 目的
对一张候选图按资产类型逐项验收，判定 accepted / rejected / rework，作为升级为 accepted ReferenceAsset 的前置门禁。**接受必须基于人工图像复核，不得由模型自动通过。**

## Required Inputs / 必需输入
- 候选图（`01-raw/story-lab/generated-raw/`）。
- GenerationRun（含 `execution_telemetry` / `actual_prompt_sent_to_external_tool`）。
- 人工填写的 `image_review_form`；合并后的 `asset_qa_result`。

## Pass Criteria / 通过标准
- [ ] 该资产类型的全部验收问题通过（R00/R01/R02/S/P01 各自清单）。
- [ ] 遥测齐全且 `actual_prompt_sent_to_external_tool` 已记录。
- [ ] 人工图像复核 `decision == pass`，证据齐全。
- [ ] 画布比例正确；`reference_dependencies` 均为 accepted。

## Fail Criteria（硬失败 / Hard Failures）
- [ ] 缺遥测或缺人工复核表单。
- [ ] `image_review_form` 出现 hard-fail（火柴人、符号表、比例错误等）。
- [ ] 绑定 rejected / deprecated 资产作依赖。

## Blocking Conditions / 阻断条件
- 未做人工图像复核 → 阻断（QA 旁路防护）。
- R00 未 accepted 时不得验收依赖它的 R01/R02/S。
- `pending` / `conditional_pass` 不得直接 accepted。

## Required Evidence / 必需证据
- `50-agent-work/story-lab/image-review-forms/<id>.json` + `50-agent-work/story-lab/qa-results/<id>.md`。

## Output Decision / 输出决策
- `accepted` → J 工作流升级 ReferenceAsset；`rejected` → 90-archive；`rework` → RepairNote 回上游重出。

## Target Layer / 落地层
- `50-agent-work/story-lab/qa-results/`（结论回填 ReferenceAsset canonical 卡）。

## Related Templates & Runtime / 关联模板与命令
- 模板：`GenerationRun`、`ReferenceAsset`。命令：`validate-image-review`、`merge-image-qa`、`qa-asset`。
