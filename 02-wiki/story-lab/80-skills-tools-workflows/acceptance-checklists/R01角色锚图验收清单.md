# R01 角色锚图验收清单 / R01 Character Anchor Acceptance Checklist

## Purpose / 目的
验收 R01 角色锚图：确认角色身份、视觉一致性与可复用性，作为角色连续性的 accepted 依赖来源。

## Required Inputs / 必需输入
- R01 候选图、对应 Character 卡（`visual_lock`、`forbidden_changes`）、人工 `image_review_form`。
- 已 accepted 的 R00（纸张笔触锚图）。

## Pass Criteria / 通过标准
- [ ] 角色身份与 Character 卡 canonical 描述一致（发型/服饰/比例/标志特征）。
- [ ] 符合 `visual_lock`，未触犯 `forbidden_changes`。
- [ ] 多姿态/表情下保持同一角色可识别。
- [ ] 儿童安全；可作为角色参考复用。

## Fail Criteria（硬失败 / Hard Failures）
- [ ] 角色身份错误或与 canonical 描述冲突。
- [ ] 携带完整剧情 / 完整场景 / 道具集合（越界——那是 R02/S 的职责）。
- [ ] 不安全内容；使用作者姓名作风格指令。

## Blocking Conditions / 阻断条件
- R00 未 accepted（R01 依赖 R00 的纸张/笔触属性）→ 阻断。
- 未做人工图像复核 → 阻断。

## Required Evidence / 必需证据
- `image_review_form` + `asset_qa_result`（`50-agent-work`）。

## Output Decision / 输出决策
- `accepted` → ReferenceAsset（`visual_role: character-anchor`）；`rework` → RepairNote；`rejected` → 90-archive。

## Target Layer / 落地层
- `50-agent-work/story-lab/qa-results/`；accepted 结论回填 `02-wiki/story-lab/reference-assets/`。

## Related Templates & Runtime / 关联模板与命令
- 模板：`Character`、`ReferenceAsset`。命令：`qa-asset`。
