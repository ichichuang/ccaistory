## runtime 自动化生产执行层

- `runtime/` 是 AI Story Compiler v0.1 的本地执行层，并包含 Skill Runtime v0.2、Story Analyzer 与 Skill Executor。
- `runtime/contracts/` 是**校验规则**（状态机、视觉资产、skill、质量门禁、pipeline action）的机器可读事实源；镜像这些规则的 Markdown 结构规范须与 contracts 一致，冲突时以 contracts 为准。
- 生产知识（故事、角色、场景、视觉风格、prompt recipe、执行包、生成记录、QA、返修）的长期 canonical 事实源是 `02-wiki/story-lab/` 的 Obsidian Markdown 卡；runtime 产物是派生缓存，持久生产决策必须回写到对应 canonical 卡。
- 新增或修改**校验规则**必须先改 contracts，再同步结构规范文档和测试。
- runtime 把状态机、技法选择、Skill Runtime、Story Analyzer、story_graph 检查、Prompt 编译、语义 Lint、执行遥测和资产 QA 转为可执行检查。
- Multimodal QA v0.1 生成结构化图像复核表单，要求人工填写后由 runtime 校验，并合并为 `asset_qa_result`。
- Pipeline Runner 是 runtime 的阶段编排层，基于 contracts 生成可规划、可阻断、可恢复、可审计的 dry-run 流水线。
- Pipeline Runner 默认 dry-run，只写 `runtime/.runs/{run_id}/run_manifest.json` 与 `runtime/.runs/{run_id}/checkpoint.json`。
- Artifact Registry 是 runtime 的**派生缓存与血缘助手**（artifact 身份、hash、血缘、状态），默认写入 `runtime/.artifacts/registry.json`；它**不是** canonical 资产登记表——canonical 资产事实源是 `02-wiki/story-lab/` 的 reference asset 卡与执行包卡。
- accepted reference asset 必须能追溯到 compiled_prompt、external_generation_candidate、execution_telemetry、image QA 和 asset_qa_result。
- rejected / deprecated asset 不得作为参考依赖。
- 不允许同名候选覆盖，不允许无 telemetry accepted，不允许无 image QA accepted，不允许无 QA accepted。
- Skill Orchestrator 负责选择 skill；如果 Story Analyzer 输出 `recommended_skill_plan`，优先使用 Analyzer 建议。
- Skill Runtime 负责评估 skill 是否真正作用到节点。
- Story Analyzer 负责内容感知诊断，包括 story type、页数、节点弱点、线索 ledger、张力曲线和角色牵引。
- Story Analyzer 高风险时，Skill Runtime/Executor 不应继续运行。
- Patch Generator 负责生成结构化修复建议；Patch Applier 默认只写入建议字段，不直接改写正文。
- Skill Executor 负责生成候选、评分、解决冲突、形成 proposed_changes；所有 proposed_changes 必须人工批准后才能应用。
- Skill Executor 默认不直接写正式字段，不允许作者风格模仿，不允许为了钩子牺牲清晰度和儿童安全。
- Analyzer 不改写内容；内容改写仍由 Skill Executor `proposed_changes` 加人工批准完成。
- Repair Loop 负责决定是否进入下一门禁。
- 新故事生产必须先经过 `python runtime/aistory.py status` 与对应 runtime 门禁。
- 技法选择后可运行 `python runtime/aistory.py check-skill-graph --graph <story_graph_json_path>`。
- 外部出图前必须通过 `compile-asset` 和 `lint-asset` / `lint-prompt`。
- 外部出图阶段必须停在人工执行点，Pipeline Runner 不调用 WebGPT 或外部出图工具。
- 图片 accepted 前必须通过 `validate-telemetry`、`validate-image-review`、`merge-image-qa` 和 `qa-asset`。
- Pipeline Runner 不替代人工审美批准，accepted/rejected 前必须有 telemetry、image QA 与 asset QA。
- 维护 contracts 时运行 `python runtime/aistory.py validate-contracts` 与 `python runtime/aistory.py check-contract-drift`。

## 硬链路

1. 故事核心
2. 页数合理性审查
3. 技法组合选择
4. Skill Runtime 检查
5. Story Analyzer 内容诊断
6. Skill Executor 候选修复与人工批准
7. story_graph 钩子强化
8. 平台图文故事编排
9. 图读测试
10. 视觉项目总清单
11. 视觉资产本体
12. 源插图 Prompt 编译
13. 源插图语义 Lint
14. 源插图试产任务清单
15. 外部出图单张执行
16. 执行遥测
17. Multimodal QA人工图像复核
18. 资产级细粒度验收
19. Artifact Registry 血缘校验
20. accepted/rejected
21. 平台发布页合成
22. 人工完整阅读
23. 发布前检查
