# 平台图文故事编排技能

状态：活动
生产体系：平台图文故事主线生产体系
唯一事实源：故事核心.json

## 目的

把 story_graph 转为 platform_story_page 队列。

## 输入

- 当前项目的 `故事核心.json`。
- 上游已通过门禁的派生视图。
- 本阶段对应的结构规范、模板和验收清单。

## 输出

- 平台图文故事编排、页序表。
- 写入或更新 `quality_gates` 所需证据。
- 需要时追加 `derived_views` 登记和操作日志。

## 读取的唯一事实源字段

- story_graph.nodes.typed_hook, typed_narration, final_hook_sentence, platform_page_layout。

## 创作者技法库接入

- 必须读取 `创作者技法库/技法总览.md`、已选组合矩阵和 `结构规范/技法增强故事图结构规范.md`。
- 平台图文故事编排时必须继承或生成 `typed_hook`、`typed_narration`、`final_hook_sentence`。
- 每页必须保留 `applied_skills`、`reader_retention_goal`、`next_page_question`、`visual_memory_point`。
- 编排输出不得删除 story_graph 的技法字段。
- 钩子评分未通过时不得继续生成下游提示词。

## 允许创建的派生视图

- 与本技能输出同名的 Markdown 报告或模板实例。
- 不得创建新的事实源；所有派生视图必须声明“唯一事实源：故事核心.json”。

## 禁止行为

- 不得让插图承载主叙事；不得让主线页只有标题。
- 不得调用 外部出图工具、生成图片、生成发布材料或改写原始资料，除非当前任务明确授权。
- 不得使用过期整页图文模式作为正式主线。

## 门禁

- 输入字段存在且与上游派生视图一致。
- 每页必须有事件推进、信息增量和下一页钩子。
- 复杂长篇完整主线必须先通过页数合理性审查。
- 完整出图前必须通过平台图文故事编排、图读测试、视觉设定和提示词巡检。

## 失败回退位置

- 回到故事图扩展与连贯性审查。

## 活动技能链

故事输入与版权改编技能 → 故事核心状态机技能 → 故事极简化与最小可行故事评估技能 → 长故事页数合理性审查技能 → 故事润色技能 → 故事图扩展与连贯性审查技能 → 平台图文故事编排技能 → 图文分镜与图读测试技能 → 视觉设定技能 → 提示词生成技能 → 提示词巡检与任务清单技能 → 出图执行技能 → 资产验收与返修技能 → 平台发布页合成技能 → 人工完整阅读技能 → 发布编排技能 → 发布后数据回收技能 → 故事废弃与实验停用技能
