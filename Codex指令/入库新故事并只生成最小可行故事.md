# 入库新故事并只生成最小可行故事

## 执行规则

- 保护原始资料，不修改参考包正文。
- 修改维护文件前必须确认存在时间戳变更记录。
- 更新 [[总控/总索引]] 与 [[总控/操作日志]]。
- 禁止 git init、commit、push、reset、clean、rebase 和破坏性命令。
- 维护目录和 Markdown 文件名使用简体中文。
- 遵守 [[总控/状态机与门禁]]；阻断门禁失败时停止，不继续下游。

## 任务

只执行版权与改编、故事极简化、最小可行故事评估、六到八页分镜和干读测试。干读失败即停止，不生成提示词。

## 故事核心状态机要求

执行前必须读取 `故事核心.json`。新故事必须先创建故事核心，不得直接创建独立分镜事实文件。分镜、干读、总览只作为派生视图生成。试产结果必须回填到故事核心的 `quality_gates` 和 `prompt_plan`。
