# 故事生产标准作业流程

状态：活动  
生产体系：平台图文故事主线生产体系  
唯一事实源：故事核心.json

runtime 执行层：`runtime/` 是 AI Story Compiler v0.1，本地执行状态机、技法选择、story_graph 检查、Prompt 编译、语义 Lint、执行遥测和资产 QA。Obsidian 文档是规范层；新故事生产必须经过 runtime 检查后才进入对应阶段。

## 主线硬链路

1. 故事核心
2. 页数合理性审查
3. 技法组合选择
4. story_graph 钩子强化
5. 平台图文故事编排
6. 图读测试
7. 视觉项目总清单
8. 视觉资产本体
9. 源插图 Prompt 编译
10. 源插图语义 Lint
11. 源插图试产任务清单
12. 外部出图单张执行
13. 执行遥测
14. 资产级细粒度验收
15. accepted/rejected
16. 平台发布页合成
17. 人工完整阅读
18. 发布前检查

## 不能跳过

故事核心、页数审查、技法组合、钩子强化、平台编排、图读测试、视觉项目总清单、视觉资产本体、Prompt 编译、语义 Lint、试产任务清单、执行遥测、资产级验收、人工完整阅读。

## 视觉链硬门禁

- 外部出图前必须通过 `python runtime/aistory.py compile-asset --spec <path>` 和 `python runtime/aistory.py lint-asset --spec <path>`；已有 compiled_prompt 时必须通过 `lint-prompt`。
- R00 不得出现人物、火柴人、完整场景、道具集合或符号散点表。
- R00 未 accepted 不得继续 R01/R02。
- 外部出图前必须有 compiled_prompt 和 semantic_lint=passed。
- 外部出图后必须有 execution_telemetry。
- 没有 actual_prompt_sent_to_external_tool 不得 accepted。
- 没有资产级细粒度验收不得 accepted。
- 图片 accepted 前必须通过 `python runtime/aistory.py validate-telemetry --telemetry <path>` 和 `python runtime/aistory.py qa-asset --qa <path>`。
- reference_dependencies 不满足不得 accepted。
- 资产 accepted 后才能解锁下游依赖资产。
