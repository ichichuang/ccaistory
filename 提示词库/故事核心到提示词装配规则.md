# 故事核心到提示词装配规则

唯一事实源：故事核心.json

## 装配顺序

1. 读取 `project_identity`、`source_status` 和 `machine_state`。
2. 读取 `story_contract`，确认主角、威胁、规则、错误行动、后果、反转和结尾回扣。
3. 读取 `story_graph.nodes` 与 `story_graph.edges`，只使用已通过门禁的节点。
4. 对每个节点装配 `typed_hook`、`typed_narration`、`final_hook_sentence`、`reader_retention_goal`。
5. 从 `visual_project_manifest` 装配角色、道具、场景、纸张、调色和参考依赖。
6. 分别输出 source_illustration、child_note_layer、typed_narration、platform_story_page。

## 字段映射

| 故事核心字段 | 提示词对象 |
|---|---|
| `story_graph.nodes.source_illustration_focus` | source_illustration |
| `story_graph.nodes.child_note_candidates` | child_note_layer |
| `story_graph.nodes.typed_narration` | typed_narration |
| `story_graph.nodes.platform_page_layout` | platform_story_page |
| `story_graph.edges.answer_in_next` | next_page_question / final_hook_sentence |
| `visual_project_manifest.paper_fingerprint` | 清洁白纸作业本锁 |

## 禁止

- 不得从聊天记忆补全事实。
- 不得使用归档任务清单作为活动提示词来源。
- 不得在源插图提示词中加入 40 字以上主叙事正文。
