# 全库巡检

状态：活动  
生产体系：平台图文故事主线生产体系  
唯一事实源：故事核心.json

## 1. 任务定位
巡检底层知识库结构、状态机、JSON、断链、空状态和门禁完整性。

成功时写入 `machine_state.current_state=repository_audit_completed`，并把 `next_allowed_action` 设为 `manual_review_or_next_allowed_action`。本指令是可执行 SOP，不依赖“按某流程执行”这类外包描述。

runtime 是自动化生产执行层；Obsidian 文档是规范层。巡检必须确认新故事生产、外部出图、图片 accepted 均受 runtime 门禁约束。

## 2. 前置状态要求
- 只读或仅更新明确授权的维护文件。
- 不得创建故事、图片、执行包、发布包、备份或复盘归档。

前置状态不满足时停止，不创建替代文件，不猜测缺失字段。

## 3. 必须读取的文件
- 总控/总索引.md
- 总控/正式维护文件清单.md
- 总控/常用指令.md
- 总控/状态机与门禁.md
- Codex指令/*.md
- 工作流/*.md
- 生产技能/*.md
- 模板/*.json
- runtime/README.md
- runtime/aistory.py
- runtime/schemas/*.json
- runtime/skill_registry/skills.json

## 4. 允许创建或更新的文件
- 总控/底层SOP可执行性自检.md（仅在授权时）

所有写入必须能从 `故事核心.json.derived_views` 或 `visual_pipeline` 回溯。

## 5. 禁止行为
- 不运行 Git 命令。
- 不创建历史故事、测试故事、图片、执行包、发布包、备份或复盘归档。
- 不调用 WebGPT、外部出图工具或任何自动批量生成图片能力，除非本阶段明确是人工外部单张执行后的回填。
- 不跳过上游门禁，不把派生视图当成第二事实源。
- 不引用历史故事名、历史 asset_id、旧项目或失败案例。

## 6. 详细执行步骤
1. 确认故事数量、图片资产数量、发布材料数量保持为 0。
2. 确认正式入口和活动生产模式存在。
3. 运行或模拟 `python runtime/aistory.py status`、`validate`、`smoke-test`。
4. 检查 runtime 是否覆盖状态机、skill registry、skill orchestrator、graph engine、prompt compiler、semantic lint、execution telemetry 和 QA engine。
5. 检查 Codex 指令是否自包含：前置、读取、写入、禁项、状态更新、回退、验证、报告。
6. 检查工作流是否有阶段顺序、状态迁移、人工检查点和下游入口。
7. 解析 JSON 模板、runtime schema、runtime fixture 并检查断链。

## 7. 质量门禁
- 发现具体故事项目或历史产物时立即停止。
- JSON 解析失败时不得报告可执行。
- 断链不为 0 时必须列出断链。

## 8. machine_state 更新规则
- 成功：写入 `current_state=repository_audit_completed`、`gate_result=passed`、`next_allowed_action=manual_review_or_next_allowed_action`。
- 需返修：保留当前上游状态，写入 `gate_result=rework_required`，并把返修位置写入 `quality_gates.<stage>.repair_target`。
- 阻断：写入 `gate_result=blocked`，保留 `next_allowed_action=resolve_blocker`。
- 每次更新必须同步记录本阶段派生视图路径或阻断原因。

## 9. blocked_actions 更新规则
- 上游未通过时必须阻断 `generate_images`、`generate_prompt_package`、`full_generation`、`publish`。
- 视觉链路未到 `pilot_task_list_ready` 前必须阻断外部出图。
- 没有 runtime 通过的 `compiled_prompt`、`semantic_lint=passed`、`execution_telemetry`、`actual_prompt_sent_to_external_tool` 和资产级 QA 时，必须阻断 `asset_accept`。
- R00 未 accepted 时必须阻断依赖 R00 的 R01/R02 和 S 页资产。

## 10. 失败回退路径
- 结构字段缺失：回退到 `故事核心状态机技能` 修复 story_core。
- 上游内容不足：回退到最近一个已通过的上游阶段。
- 视觉本体或 prompt 出错：回退到视觉资产本体或 Prompt 编译，不直接改候选图。
- 外部执行事实缺失：回退到执行遥测补录；无法补录时标记 blocked。

## 11. 最终验证
- 本阶段允许写入的文件存在且可追溯。
- JSON 文件可解析。
- `machine_state`、`blocked_actions`、`quality_gates` 与本阶段结论一致。
- 没有新增故事外产物、图片、执行包、发布包、备份或复盘归档。
- 下游入口只开放 `manual_review_or_next_allowed_action`。

## 12. 返回报告格式
- 是否完成：是/否
- 当前状态：repository_audit_completed
- 下一允许动作：manual_review_or_next_allowed_action
- 已读取文件：列出实际读取路径
- 已创建或更新文件：列出实际路径
- 门禁结果：passed/rework_required/blocked
- blocked_actions：列出当前阻断动作
- 失败回退位置：如无则写“无”
- 残余风险：如无则写“无”
