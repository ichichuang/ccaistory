# 唯一事实源验收清单

## 必查项

- [ ] `故事核心.json` 存在且 JSON 可解析。
- [ ] `machine_state.current_state` 使用枚举值。
- [ ] `machine_state.gate_result` 使用枚举值。
- [ ] 主线 `story_graph.nodes` 至少覆盖全部主线页。
- [ ] 相邻主线页均有 `story_graph.edges`。
- [ ] 角色、道具、地点合同在故事核心中统一定义。
- [ ] 派生视图顶部声明唯一事实源。
- [ ] 提示词计划引用故事图和视觉项目合同。
- [ ] 未通过试产前阻断完整出图和发布。

## 硬失败条件

1. 同一角色事实在两个文件中不同。
2. 同一道具有多个规范名称。
3. 项目状态是自然语言长句。
4. 分镜节点没有 `state_change`。
5. 干读测试保存了独立剧情事实而非派生自故事图。
6. 项目总览与故事核心状态冲突。
7. 提示词包没有引用故事核心中的节点。
8. 外部出图工具 任务清单不是从已通过的 `prompt_plan` 派生。
9. 未通过 pilot gate 却生成完整任务清单。
10. 派生视图未声明唯一事实源。
