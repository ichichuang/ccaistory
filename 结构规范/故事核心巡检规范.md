# 故事核心巡检规范

## 巡检输入

- `故事核心.json`
- `derived_views` 中列出的 Markdown 文件
- 总控索引、生产技能、工作流、验收清单和 Codex 指令

## 巡检项目

- JSON 可解析，顶层字段完整。
- `machine_state.current_state` 和 `gate_result` 均为允许值。
- 主线 `story_graph.nodes` 连续，且每个节点有 `state_change`。
- 相邻主线节点有 `story_graph.edges`。
- 角色、道具、地点规范名只在故事核心定义。
- 派生视图顶部含唯一事实源声明。
- 提示词包和任务清单引用 `story_graph.nodes` 与 `visual_project_manifest`。
- 未通过 `pilot_qa` 前不得生成完整任务清单或发布材料。

## 硬失败

- 项目状态是自然语言长句。
- 同一角色事实在两个文件中不同。
- 同一道具有多个规范名称。
- 干读测试保存独立剧情事实而非派生自故事图。
- 外部出图工具 任务清单不是从已通过的 `prompt_plan` 派生。
